## Slide 2: Introduction

**Headline:** What is this ERP System?

**Key Points:**
- **Problem:** Managing employees, inventory, and sales manually is inefficient and error-prone.
- **Solution:** A centralized, digital platform to handle all core business functions.
- **Goal:** Increase productivity, ensure data accuracy, and provide real-time insights for better decision-making.

---

## Slide 3: Key Features (Overview)

**Headline:** All-in-One Business Management

**Visual:** Icons representing Employees, Inventory, Sales, and Analytics.

**Key Modules:**
1.  **Employee Management:** Onboarding, profiles, and roles.
2.  **Inventory Control:** Stock tracking and low-inventory alerts.
3.  **Sales Tracking:** Record transactions and monitor revenue.
4.  **HR Functions:** Leave requests, attendance marking, and task assignments.
5.  **Admin Dashboard:** High-level metrics and reports.

---

## Slide 4: User Roles & Access

**Headline:** Secure & Tailored Experience

**1. Admin (The Manager)**
- Full control over the system.
- Can add/edit employees, inventory, and departments.
- Reviews leave requests and assigns tasks.
- Views comprehensive analytics.

**2. Employee (The User)**
- Personalized dashboard.
- Mark attendance and apply for leave.
- View and update assigned tasks.
- Access personal profile and announcements.

## Slide 5: Technical Architecture (The Stack)

**Headline:** Built with the MERN Stack

**Frontend:**
- **React 19 + Vite:** Fast, modern UI.
- **Tailwind CSS 4:** Responsive and clean design.
- **Redux / Context (if applicable):** State management.

**Backend:**
- **Node.js + Express:** Robust and scalable API.
- **MongoDB + Mongoose:** Flexible NoSQL database.
- **JWT + Bcrypt:** Secure authentication and data protection.

## Slide 6: Database Schema (Brief)

**Headline:** How Data is Organized

- **Users:** Stores employee info, roles, and credentials.
- **Inventory:** Products, stock levels, and categories.
- **Leaves:** Leave requests, status (Pending/Approved), and dates.
- **Tasks:** Task details, assigned user, deadline, and priority.
- **Sales:** Transaction records associated with inventory items.
